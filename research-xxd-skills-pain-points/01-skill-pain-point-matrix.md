# 10 个 xxd skills 的痛点矩阵

这次改写的核心判断：`xxd-*` 不是 10 个同款配色模板，而是 10 个不同设计失败根因的工具。每个 skill 都应先识别自己的问题边界，再输出对应的工作流。

| Skill | 目标用户场景 | 真实痛点 | 这次改写方向 |
| --- | --- | --- | --- |
| `xxd-color-brief` | 项目刚开始，只有情绪词、参考图、客户反馈。 | 模糊形容词无法直接变成色板；如果过早选色，后续会反复争论方向。 | 先把形容词翻译成冷暖、明度、饱和度、对比和风险，再给起始色与下一步 skill。 |
| `xxd-palette-builder` | 已有锚点色、HEX、情绪或场景，需要实际 palette。 | 颜色选择过多，swatches 好看但缺少主辅点缀和比例。 | 按可靠、识别、对比三种策略建 palette，强制先分角色再展示颜色。 |
| `xxd-palette-applier` | 已有 palette，要放进海报、网页、PPT、包装或 UI。 | 色板离开 swatches 后，背景、标题、按钮、装饰互相抢。 | 按具体载体建立阅读路径、面积比例、角色地图和误用清单。 |
| `xxd-ui-token` | 传统色要进入产品界面、网站或设计系统。 | 直接交 HEX 会导致硬编码、深色模式混乱、组件状态不一致。 | 建立 primitive、semantic、component、mode 四层 token，默认输出 CSS variables。 |
| `xxd-accessible-color` | 某些颜色要用于正文、按钮、状态或图表。 | 好看的传统色可能读不清；状态和图表经常只靠颜色表达。 | 用 WCAG ratio 做判断，失败时从 742 色库找修复色，并补非颜色线索。 |
| `xxd-brand-system` | 颜色要支撑长期品牌，而不是一次物料。 | 不同渠道和活动会让品牌色漂移；缺少禁用规则和治理。 | 从类别、差异、价格带、渠道压力出发，输出锚点、支撑、比例、禁用和治理规则。 |
| `xxd-data-viz` | 做分类图、连续色带、发散图、地图或仪表盘。 | 海报配色不等于图表配色；数据需要区分、排序、中点和可访问性。 | 先判定 categorical/sequential/diverging/highlight/semantic，再选传统色。 |
| `xxd-existing-design-audit` | 有旧截图、CSS、Figma styles、品牌色或 HEX 清单。 | 旧稿常有重复近似色、角色混乱、低对比和品牌识别风险。 | 先库存盘点，再诊断根因，提供保守、均衡、全系统三种修复层级。 |
| `xxd-content-series` | 小红书、公众号、视频、课程、栏目等长期内容。 | 每期都换色会丢识别；完全固定又容易疲劳。 | 拆成固定层、变量层、模板层和轮换规则，让系列既统一又有变化。 |
| `xxd-print-packaging` | 包装、书籍、文创、标签、礼盒、实体材质。 | HEX 不能保证印刷效果；材质、CMYK、spot、proof 和货架环境会改变颜色。 | 以材质和生产风险为主线，输出面板计划、系列延展和打样检查清单。 |

## 对 skills 的共性约束

- 所有推荐色必须能追溯到 `docs/chinese-color-master-list.md` 和 `docs/chinese-color-harmony.csv`。
- 如果用户输入外部 HEX，必须标注为“映射到最接近传统色”，不能当成新源色。
- 输出要解决设计动作，不只解释色彩理论。
- 能给具体规则时，不用泛泛的审美判断。
