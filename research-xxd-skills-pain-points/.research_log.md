# 研究日志

## 2026-06-09 - 逐个重构 xxd skills

- 用户问题：这些 skills 要依次联网找对应痛点，一个个针对性开发，不能套模板。
- 本轮目标：为 10 个已存在的 `xxd-*` skills 建立差异化痛点依据，并落实到 `SKILL.md` 工作流。
- 已读旧文件：10 个 `skills/xxd-*/SKILL.md`、README skills 表格、首页 skills 展示区。
- 新增来源：
  - [Nielsen Norman Group - Mood Boards in UX](https://www.nngroup.com/articles/mood-boards/) | 实战方法 | 支撑 `xxd-color-brief` 的视觉方向对齐问题。
  - [Atlassian Design System - Color](https://atlassian.design/foundations/color/) | 一手规范 | 支撑角色化用色、UI 层级和状态色。
  - [Atlassian Design Tokens](https://atlassian.design/tokens/design-tokens/) | 一手规范 | 支撑 `xxd-ui-token` 的语义 token 和交付逻辑。
  - [Design Tokens Format Module](https://tr.designtokens.org/format/) | 一手规范 | 支撑 token 分层和跨工具交付。
  - [Figma Variables Guide](https://help.figma.com/hc/en-us/articles/15339657135383-Guide-to-variables-in-Figma) | 一手文档 | 支撑 mode、变量、设计系统交付。
  - [W3C WCAG Contrast Minimum](https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html) | 一手规范 | 支撑对比度阈值。
  - [W3C WCAG Use of Color](https://www.w3.org/WAI/WCAG22/Understanding/use-of-color.html) | 一手规范 | 支撑不要只用颜色传递信息。
  - [ColorBrewer Schemes](https://colorbrewer2.org/learnmore/schemes.html) | 一手工具说明 | 支撑分类、顺序、发散图表色逻辑。
  - [IBM Carbon Data Visualization Color Palettes](https://carbondesignsystem.com/data-visualization/color-palettes/) | 一手规范 | 支撑图表 palette 的可读性与分层。
  - [Pantone - Understanding Different Color Spaces](https://www.pantone.com/articles/color-fundamentals/understanding-different-color-spaces) | 一手文档 | 支撑 CMYK、spot/Pantone 与设备色空间差异风险。
- 减少的不确定性：skills 应按 color brief、palette choice、placement、token、accessibility、brand governance、data encoding、audit/migration、content cadence、physical production 十个不同问题建模。
- 判断变化：不再把 10 个 skills 作为同一套配色流程的不同阶段，而是作为不同设计失败根因的工具。
- 写入/更新文件：10 个 `skills/xxd-*/SKILL.md`，并同步更新网站与 README 中的 skill 介绍。
- 仍待验证：真实使用中各 skill 是否还需要示例输入和测试 prompts。
